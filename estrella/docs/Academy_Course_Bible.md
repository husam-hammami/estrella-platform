# Estrella Academy Course Bible

## Current Direction Override

The course promises and course outcomes in this document remain useful.

The heavy dossier/workspace presentation is superseded by:

`estrella/docs/Academy_Simple_Mode_Brief.md`

For implementation, simplify every course into one idea, one choice, and one visible win per screen.

This is the source of truth for Academy course material, interaction model, and UI intent.

An implementer must not invent course structure from screenshots. They must build from this document, `academy-content.js`, and the approved visual handoff.

## Academy Thesis

Estrella Academy is not a course library.

It is a premium career artifact desk. Each course gives the learner one practical artifact they can use immediately: a plan, script, map, playbook, or operating sheet.

The UI must make the learner feel:

- I know what I am building.
- I know the next small move.
- My work is being captured into something useful.
- This feels polished and adult, not school-like.

## Implementation Contract

Do not render lessons as:

- long article pages;
- generic paragraphs plus buttons;
- LMS sidebars;
- empty panels;
- detached chat widgets;
- generic course cards;
- fake gamified reward screens.

Render every lesson as:

- a dossier page;
- a course-specific instrument;
- one saved artifact piece;
- quiet Estrella guidance in the margin.

## Universal Lesson Studio

Each lesson has five possible beats. These are renderer states, not learner-facing jargon.

| Beat | Purpose | UI |
|---|---|---|
| Overview | Why this lesson matters | left page with hook and artifact promise |
| Method | The one idea | short paper section, not full article dump |
| Instrument | Learner action | course-specific instrument on right page |
| Check | Confirm understanding | 1-2 compact checks |
| Saved | Show progress | artifact piece slides into plan/vault |

The active screen must never be mostly empty. If the learner is choosing, the instrument must be visibly active. If the learner has saved, the artifact piece must be visibly captured.

## Interaction Primitives

Use only these primitives unless a course bible section says otherwise:

- **Choose**: pick one move, label, quadrant, or card.
- **Sort**: place a scenario into a board, map, lane, or file.
- **Rehearse**: pick or refine a spoken move.
- **Check**: answer one focused confirmation question.
- **Save words**: write only the final artifact language.
- **Review**: compare saved pieces and choose what to keep.

Writing is not the default. Writing appears only when the artifact itself needs words.

## 1. Confidence Reset

### Course Promise

The learner can notice a confidence drop, name the trigger, answer the critic, collect proof, and follow a simple seven-day reset plan.

### Final Artifact

**Confidence reset plan**

Sections:

- Trigger map
- Critic answer slips
- Evidence file
- Rejection reframe
- Reset ritual
- Seven-day plan

### Instrument

**Mirror Journal**

The UI is a private journal spread with pinned slips. It should feel calm, intimate, and precise. The learner is not typing therapy notes. They are building a practical reset plan.

### Lesson Pieces

| Lesson | Artifact Piece | Interaction |
|---|---|---|
| Why confidence collapses | Trigger map | choose the honest trigger |
| The inner critic toolkit | Critic answer slip | choose the better response |
| Mapping your win history | Evidence file card | collect proof |
| Reframing the rejection | Reframe card | choose the useful reading |
| Building a confidence ritual | Reset ritual card | choose the ritual move |
| Your 7-day plan | Seven-day plan | assemble the plan |

### UI Behavior

The right page should show journal slips:

- Name the trigger
- Answer the critic
- Collect proof
- Use the ritual

When selected, the slip physically joins the reset plan area. Do not show four generic pill buttons.

### Estrella Behavior

Estrella should sound calm and direct:

- "Name what happened before you judge yourself."
- "Pick the trigger that is most honest, not the one that sounds best."
- "Save proof, not praise."

She should not diagnose, over-comfort, or ask for long emotional writing.

## Acceptance Gate

Reject the build if:

- the screen works without the course instrument;
- replacing the course name would make the page still make sense;
- saved learner work is invisible;
- the right page is empty;
- the UI feels like a worksheet;
- the UI uses generic list rows for the main interaction;
- the learner cannot tell what artifact they are building within three seconds.
